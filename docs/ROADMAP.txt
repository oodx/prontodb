ProntoDB Development Roadmap

Phase 1: Basic Working System

1. Foundation Infrastructure
   1.1 XDG path management and verification
   1.2 XDG_HOME switchable path support
   1.3 Complete dispatcher flag parsing enhancement
   1.4 Address validation for basic project.namespace.key structure
   1.5 Direct CRUD dispatcher routing setup

2. Flag and Dot Addressing
   2.1 Flag addressing (--project, --namespace, --key)
   2.2 Dot addressing (project.namespace.key parsing)
   2.3 Delimiter override support
   2.4 Address validation helpers

3. Configuration System
   3.1 Config files and config flag
   3.2 Database switching with --select flag
   3.3 Sticky cursor files for database selection persistence
   3.4 Per-user cursor files (--user flag support)
   3.5 Multi-user/multi-instance support

Phase 2: Core Object CRUD (Isolated)

4. Database Management
   4.1 Database CRUD operations (create, switch, rename, delete)
   4.2 Database selection and persistence
   4.3 Database-config integration

5. Table Management
   5.1 Table CRUD operations (create, drop, list, describe)
   5.2 Schema management functions
   5.3 Query helper functions
   5.4 Table-database relationship management

6. Project Management
   6.1 Project CRUD operations (create, drop, list, describe)
   6.2 Project-to-table mapping
   6.3 Project metadata management
   6.4 Project-database relationship management

7. Namespace Management
   7.1 Namespace CRUD operations (create, drop, list, describe)
   7.2 Namespace-to-table mapping
   7.3 Namespace metadata management
   7.4 Namespace-project relationship management

8. Key Operations
   8.1 Key CRUD operations (set, get, delete, scan, list)
   8.2 Key operations using full object hierarchy
   8.3 Key-namespace relationship management

9. Metakey Operations
   9.1 Metakey CRUD operations
   9.2 Metakey management and relationships

Phase 3: Integration

10. User Command Integration
    10.1 Traditional set/get/del commands
    10.2 Address resolution through object hierarchy
    10.3 Flag integration with all objects
    10.4 Error handling and exit codes

11. Admin Command System
    11.1 Admin dispatcher and command routing
    11.2 Database admin commands
    11.3 Table admin commands
    11.4 Project admin commands
    11.5 Namespace admin commands

12. Lifecycle Operations
    12.1 Install command (database and table setup)
    12.2 Uninstall command (cleanup operations)
    12.3 Backup command (export functionality)
    12.4 Restore command (import functionality)

Phase 4: Advanced Features

13. Discovery Commands
    13.1 Projects listing command
    13.2 Namespaces listing command
    13.3 Basic admin commands

14. TTL and Caching
    14.1 TTL namespace support
    14.2 create-cache command
    14.3 TTL expiration logic
    14.4 sys_namespaces metadata
    14.5 Cache management commands

15. Stream Processing
    15.1 Token parser
    15.2 Auth preamble handling
    15.3 Transaction boundaries
    15.4 Security enforcement
    15.5 Stream format support

16. Advanced Operations
    16.1 Scanning and listing with prefix filtering
    16.2 JSON output support
    16.3 Import/export TSV format
    16.4 Roundtrip validation

17. Error Handling and Testing
    17.1 Comprehensive error handling
    17.2 Exit code management
    17.3 Test framework setup
    17.4 Validation tools

18. Concurrency and Performance
    18.1 busy_timeout configuration
    18.2 WAL mode setup
    18.3 Multi-writer safety
    18.4 Proper locking and coordination
